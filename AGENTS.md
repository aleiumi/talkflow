# AGENTS.md — TalkFlow 开发规范

## 项目概览
TalkFlow 是一个网页版社交话题引导工具，核心目标是帮助用户自然开启对话、延续话题、深化情感连接。

## 项目结构
`
E:\\TalkFlow\\
├── 任务说明.md        # 完整需求文档
├── 进度记录.md        # 里程碑与进度追踪
├── AGENTS.md          # 本文件：Agent 开发规范
├── index.html         # 入口文件
├── css\\              # 样式文件
├── js\\               # 脚本文件
├── data\\             # 话题数据（JSON）
└── assets\\           # 图片、图标等资源
`

## 开发原则
1. **轻量优先**：初期不引入重型框架，用原生 HTML/CSS/JS 快速验证 MVP
2. **移动端优先**：所有设计以手机端为第一目标，再适配桌面端
3. **数据与代码分离**：话题数据存为 JSON，方便后期扩展和后台接入
4. **逐步迭代**：先做 MVP（话题盲盒 + 话术引导），再做高级功能

## 编码规范
- 文件命名：小写 + 连字符，如 	opic-blank.js
- 变量命名：驼峰命名法
- 注释：关键逻辑加中文注释
- 每个功能模块单独一个 JS 文件

## 开发顺序
按以下顺序实现：
1. 项目脚手架（index.html + 基础 CSS 布局）
2. 话题数据结构设计（data/topics.json）
3. 话题盲盒（核心）
4. 话术+思维双引导展示
5. 维度筛选
6. 救急模式
7. 话题实验室
8. 响应式 & PWA

## Agent 工作指引
- 每次改动前检查 进度记录.md，确认当前阶段
- 完成一个里程碑后，立即更新 进度记录.md
- 需求不明确时，参考 任务说明.md，如仍有疑问再询问用户
- 代码风格与本项目约定保持一致
